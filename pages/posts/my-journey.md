Hey, I’m Hem Bahadur Pun, a front-end engineer based in Pokhara, Nepal. Over the years, I’ve explored various parts of web development and built projects that reflect my journey. Today, I want to share how I got started, what I've learned along the way, and where I see myself going.

## How it started

My journey into coding started like many others — with curiosity and a desire to create something meaningful. One of my first memorable projects was [my_first_react_project](https://github.com/hempun10/my_first_react_project), a simple movie application built using React and the OpenMovie API. It was experimental, a way for me to understand how React worked, how APIs could be connected, and the power of JavaScript frameworks.

Starting small and focusing on practical applications helped me build confidence. I firmly believe that hands-on projects are the fastest way to solidify learning.

## Open source and personal projects

Contributing to and creating personal projects has been an invaluable part of my growth as a developer. Open source especially taught me the importance of writing clean, readable code and thinking about long-term maintainability.

For instance, when I created [devdaily](https://github.com/hempun10/devdaily), my primary goal was to experiment with TypeScript while building something useful. It pushed my understanding of type safety and frontend tooling. Another project that reflects my recent approaches is [Gitfolio](https://github.com/hempun10/Gitfolio), which showcases repositories dynamically — taking customization to the next level.

These projects are not just about code; they’re also opportunities to stay curious, break things, fix them, and share progress with the community.

## Lessons along the way

- **Focus on fundamentals:** While working with modern frameworks is exciting, I’ve realized it’s the fundamentals — HTML, CSS, and vanilla JavaScript — that hold everything together.
- **Iteration is key:** Every project I’ve worked on, including [devdaily](https://hempun10-devdaily.mintlify.app/) and [movie application](https://moviemandu.netlify.app/), has undergone multiple iterations. I’ve learned to embrace feedback and continually improve.
- **Collaboration matters:** Open source collaboration helps you learn how others approach problems, which is invaluable as a developer.

## What's next

The web is evolving fast, and I’m excited about emerging technologies and tools that simplify how developers work. Right now, I’m particularly interested in:

- Exploring deeper integrations of AI-powered workflows to accelerate development.
- Building more scalable, interactive user experiences using tools like TypeScript and React.
- Sharing my learnings through open source to help other developers.

There’s so much to look forward to, and I feel like I’ve only scratched the surface of what’s possible in frontend development.

## Wrapping up

Thanks for taking the time to read about my journey! If you’d like to connect, ask questions, or collaborate, feel free to reach out on [GitHub](https://github.com/hempun10). Let’s build an awesome web together!
